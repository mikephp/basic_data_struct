Config — Proxy
Well, profile has been created. No we should config our proxy, which will be used for traverse. You should to know five things: address of proxy, port, type, login and password.
Proxy addressIP or domain name
Portport where proxy is working
TypeSOCKSv4, SOCKSv5 or HTTP(S) proxy
Login/passwordIf proxy requiring authorization
Open the menu File->Settings. The "Default SOCKS server" tab page should be opened.
For add new proxy in the list you should to do few steps:
1Write proxy address and port into the fields.
2Select the proxy type from group boxes
3If proxy requires authorization mark the checkbox "Authorization required"
4Press the button "Add"
In most cases you need only one proxy, so this step of configuration is completed. If you want to use 'proxy chaining' you can build proxy chain Please remember: when you're adding the HTTP proxy you should to know that's it supports the CONNECT method. Typical mistake of users — add its corporate cache proxy (More...).
The 'Up/Down' buttons are for the changing order of the proxies in the chain.
The 'Check Proxy' button just checks that this port is alive and according to result mark or unmark the checkbox near proxy in the chain list. 'Proxy check' doesn't check the proxy type or entered correctly login/pasword. For now.
